Discovery

1
RSS Polling & Gap Detection source_enumerator.py
Poll RSS feeds — 15 most recent videos per tracked channel (free, no auth)
Compare RSS IDs against analysis_status table. No overlap = gap detected
Backfill via YouTube Data API v3 publishedAfter cursor (50/page, max 1000 videos)
Write new entries to batch_status.sqlite as status='pending'
RSS → Gap Detection → YouTube Data API → batch_status.sqlite

Ingest

2
Entry Points
Industrial (/yt-nlm) — nlm_batch.py creates notebooks with up to 300 sources, fetches raw transcripts via nlm source content (no credits consumed). ~18K videos/hr, 99% SNR.
Surgical (/yt-channel fetch) — transcript.py single-video fetch for targeted requests. ~720 videos/hr, lower SNR.
Both paths feed the same fetch_transcript_chain() escalation ladder
batch_status.sqlite → /yt-nlm | /yt-channel fetch → escalation chain
3
Escalation Chain transcript.py
Stage 1
yt-dlp + curl-cffiWEB client, TLS impersonation, ~5s
↘
Stage 2
yt-dlp + Firefox cookiesEJS challenge resolver for bot-checked / age-restricted videos
↘
Stage 3
NotebookLM Industrial BatchUp to 300 src/notebook, 15 parallel workers, AuthRateLimiter + CookieFreshnessTracker
↘
Stage 4
Selenium FirefoxHeadless browser, dirty output (~40% SNR from page noise)
↘
Stage 5
faster-whisperAudio transcription, medium model, CPU, int8 quantization
↘
Stage 6
youtube-transcript-apiDirect captions API, prefers non-generated English
Circuit breaker: 3 consecutive 429s → 5-minute source cooldown. Jitter: random.uniform(2.0, 10.0) * min(2^count, 32). YTIS_WHISPER_ENABLED env var controls Stage 5 (default: true).

Cache

4
Transcript Cache
Every successful escalation stage writes to transcripts.sqlite
Keyed by video_id : lang : source, WAL mode — shared across all terminals
Analysis layer checks has_cached_transcript() before selecting a provider
Escalation → transcripts.sqlite → cache lookup on analysis

Analysis

5
Provider Selection orchestrator.py
Cache first: if transcript already cached, route directly to TranscriptProvider (zero API cost)
Build candidates: local_model → gemini_sdk → ocr_clip → transcript
Failure-aware reorder: per-channel provider_score table adjusts priority based on historical success rate
Instantiate the first available provider — not hardcoded, adapts per channel
transcripts.sqlite → cache check → select_provider() → failure-aware routing
6
Provider Execution
💻 local_model
LM Studio local inference. No API cost, no network required.
🤖 gemini_sdk
Full multi-modal analysis via bin/csf-analyze. Quota-tracked.
👁 ocr_clip
Captures frames, runs OCR for code snippets + CLIP for visual tags, then summarizes.
📄 transcript
Calls fetch_transcript_chain() then summarize() via Gemini CLI LLM.
Provider → VideoAnalysisResult → mark complete in batch_status
